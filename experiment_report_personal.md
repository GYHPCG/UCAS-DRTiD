# CrossFiT 模型复现与改进实验报告

---

## 摘要

本报告记录了对 CrossFiT（Cross-Field Transformer）模型的复现与改进工作。在华为 Ascend NPU 平台上完成了模型的完整迁移，使用 DeepDRiD 数据集进行训练和评估。在此基础上，本文提出了一种基于视野重要性加权的非对称信息处理方法（View Gating），并通过多种视盘定位策略对比和模型集成实验，系统性地探索了模型性能的提升空间。最终，论文原方法（阈值分割视盘定位）在测试集上取得了 0.932 的二次加权 Kappa，本文提出的 View Gating 方法在特定配置下将 Kappa 提升至 0.923，Snapshot Ensemble 策略将测试准确率提升至 85.0%。

---

## 一、引言

糖尿病视网膜病变（DR）是导致工作年龄人群视力丧失的主要原因之一。CrossFiT 模型（BIBM 2022）提出了一种跨视野 Transformer 架构，通过融合以视盘为中心和以黄斑为中心的两种眼底图像，实现 DR 严重程度的自动分级。

本工作的主要贡献包括：

（1）**NPU 平台适配**：将原始 CUDA 实现完整迁移至华为 Ascend NPU，解决了设备调用、数据加载、位置编码等一系列兼容性问题。

（2）**多策略视盘定位对比**：系统对比了深度学习检测（不同分辨率输入）和论文阈值分割两种视盘定位方法，发现简单阈值方法反而优于深度学习方法。

（3）**非对称信息处理改进**：针对原模型对两个视野一视同仁的不足，提出了 View Embedding 和 View Gating 两种改进方案，显式建模临床中黄斑视野更重要的先验知识。

（4）**模型集成探索**：采用 Snapshot Ensemble 策略，通过集成不同训练阶段的模型快照提升分类性能。


## 三、数据集

### 3.1 DeepDRiD 数据集

本实验使用 DeepDRiD（Deep Diabetic Retinopathy Image Dataset）数据集，来源于 ISBI 2020 糖尿病视网膜病变检测挑战赛。数据集包含来自 400 位患者的 1600 张眼底图像，每位患者提供 4 张图像：左眼和右眼各两种视野（以视盘为中心和以黄斑为中心）。

| 子集 | 患者数 | 图像对数 | 图像数 |
|------|--------|---------|--------|
| 训练集 | 300 | 593 | 1186 |
| 验证集 | 100 | 200 | 400 |
| 比赛测试集 | 100 | 200 | 400 |

### 3.2 数据预处理

训练阶段：随机裁剪至 512×512、随机颜色抖动、随机灰度化（概率 0.2）、随机仿射变换。

验证/测试阶段：缩放至 384×384，归一化参数为 DeepDRiD 数据集的统计均值和标准差。

### 3.3 视盘坐标检测

由于 DeepDRiD 数据集未提供视盘标注，本工作实现了两种视盘定位方法：

**方法一：深度学习检测**。使用 fundus-image-toolbox 工具包中的 EfficientNet-B3 模型，在 NPU 上完成 1600 张图像的推理。分别测试了缩放至 512×512 和原始分辨率两种输入方式。

**方法二：论文阈值分割**。按照原论文方法，对以视盘为中心的视野直接设定原点 (0, 0)，对以黄斑为中心的视野通过亮度阈值分割定位视盘。具体步骤：将图像转为灰度图，在圆形掩码内取亮度 99.5 百分位作为阈值，通过形态学开运算去除噪点，取最大连通区域的质心作为视盘位置。

## 四、方法

### 4.1 原始 CrossFiT 模型

CrossFiT 模型由三个核心模块组成：

（1）**特征提取**：ResNet-50 骨干网络，两幅图像共享权重，输出 2048 通道特征图，通过 1×1 卷积降维至 1024 维。

（2）**跨视野注意力（CFA）**：生成特征掩码过滤背景，结合正弦余弦位置编码和视盘坐标实现空间对齐，3 层 Transformer Encoder 实现跨视野信息交互。

（3）**分类**：Max Pooling + LayerNorm + 全连接层输出 5 类 DR 等级。

### 4.2 非对称信息处理改进

原论文的 Cross-Field Attention 对黄斑图和视盘图一视同仁，但在眼底临床中，它们的信息量极度不对称——决定 DR 评级的微动脉瘤和渗出物绝大多数都在黄斑区，视盘更多是解剖参照。为此，本文提出两种改进方案：

**View Embedding**：在 Transformer 输入前，为两个视野分别添加可学习的视图嵌入向量（类似 BERT 的 segment embedding），使模型能够区分来自不同视野的 token。

**View Gating**：在特征融合阶段，添加可学习的视野重要性权重，通过 softmax 归一化后对两个视野的特征进行加权融合。初始权重设为 [0.3, 0.7]（偏重黄斑视野），让模型自动学习最优的视野权重分配。

```python
# View Gating 实现
self.view_importance = nn.Parameter(torch.tensor([0.3, 0.7]))

# 前向传播中
view_weights = F.softmax(self.view_importance, dim=0)
out1_weighted = out1 * view_weights[0]  # 视盘视野
out2_weighted = out2 * view_weights[1]  # 黄斑视野
```

### 4.3 Snapshot Ensemble

采用 Snapshot Ensemble 策略，将训练过程中不同 epoch 的模型快照进行集成。在测试时，加载多个 checkpoint，对每个样本的 softmax 概率取平均，得到最终预测。

## 六、结论

本工作在华为 Ascend NPU 平台上成功复现了 CrossFiT 模型，并通过多种改进策略探索了性能提升空间。主要发现如下：

（1）**NPU 适配可行**：CrossFiT 模型可顺利从 CUDA 迁移至 Ascend NPU，主要修改为设备调用替换和动态位置编码计算，模型功能和性能不受影响。

（2）**简单方法更优**：论文的简单阈值分割视盘定位方法（测试 Kappa 0.932）优于深度学习检测方法（0.912），验证了"简单即有效"的原则。

（3）**非对称处理有效但有限**：View Gating 方法在特定配置下可提升 Kappa（0.923 vs 0.912），但与阈值分割方法存在冲突，说明两种改进方向可能存在内在矛盾。

（4）**集成提升准确率**：Snapshot Ensemble 将测试准确率提升至 85.0%，但 Kappa 未超过最佳单模型，表明小数据集上单模型的泛化能力已接近上限。

（5）**训练稳定性待改进**：训练过程中存在明显的性能震荡，建议引入 Early Stopping 和学习率预热策略。
